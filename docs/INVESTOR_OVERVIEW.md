# Footchibol — סיקור מלא לפרויקט ולהצגה למשקיעים

**גרסה:** 1.0  
**עדכון אחרון:** מרץ 2025  
**סלוגן:** "מקפיצים ביחד" / Beach Footvolley Club

---

## 1. תמצית מנהלים (Executive Summary)

**Footchibol** היא אפליקציית מובייל **בעברית (RTL)** שמחברת שחקני **פוטוולי חופים** בישראל. המשתמשים מגלים "מעגלים" (משחקי איסוף פתוחים) לפי רמה, תאריך ומיקום, מצטרפים או יוצרים משחקים, מתקשרים בצ'אט עם המשתתפים ומדרגים שחקנים אחרי המשחק.  
הפרויקט כולל **אפליקציית Expo (React Native)** ל-iOS/Android/Web, **בקאנד NestJS**, ו-**Firebase** (Auth, Firestore, Storage, התראות).

**פיץ בשורה אחת:**  
פלטפורמה דיגיטלית ראשונה מסוגה בישראל לארגון והצטרפות למשחקי פוטוולי חופים — עם דירוג שחקנים, צ'אט קבוצתי והתראות למנהלי מעגלים.

## 2. הרעיון והחזון

### הבעיה
- שחקני פוטוולי חופים מתקשים **למצוא קבוצה ברמה מתאימה** ובזמן נוח.
- ארגון משחקים מתבצע היום ב-**וואטסאפ וקבוצות פייסבוק** — ללא סינון רמה, מיקום או תאריך מסודר.
- אין **היסטוריה ודירוג** — קשה להעריך מי מגיע, מי ברמה טובה ומי "חוטף" (no-show).

### הפתרון
- **מעגלים (Circles):** משחק פתוח עם מיקום, תאריך/שעה, רמה נדרשת (מתחיל/בינוני/מתקדם), הגבלת מגדר, סוג מגרש (חוף/אספלט/דשא).
- **גילוי חכם:** סינון לפי רמה ותאריך, "מעגלים לידך" לפי עיר/אזור, דגל "דחוף" למעגלים שצריכים שחקנים.
- **אחרי המשחק:** דירוג 1–5 בין המשתתפים — בונה **דירוג ושם** לכל שחקן (rating, אחוז נוכחות, mental score).
- **תפקיד מנהל:** מנהלי מעגלים מקבלים **התראות מותאמות** (מעגלים קרובים, תזכורות, צ'אט, מזג אוויר, מעגלים דחופים).

### חזון ארוך טווח
להפוך ל-**הפלטפורמה המובילה** לחיבור שחקני פוטוולי (וחובבי ספורט חופים) בישראל, ולהיות בסיס להרחבה גאוגרפית ותוכנית (ענפי ספורט נוספים, טורנירים, שותפויות עם מגרשים/רשויות).

| **פרופיל** | כרטיס שחקן | תמונה, רמת משחק, סטטיסטיקות (דירוג, מעגלים שהצטרף/יצר, no-shows, אחוז נוכחות, "ציון מנטלי") |

## 4. סטק טכנולוגי

| שכבת | טכנולוגיה |
|------|-----------|
| **קליינט** | Expo SDK ~55, React Native, expo-router (ניווט לפי קבצים), iOS / Android / Web |
| **state** | Zustand (auth, circles), React Hook Form + Zod לטפסים |
| **אימות** | Firebase Auth — אימייל/סיסמה + טלפון כמזהה |
| **DB** | Cloud Firestore — `users`, `circles`, `chats/{circleId}/messages`, `ratings` |
| **אחסון קבצים** | Firebase Storage — תמונת פרופיל, סרטון |
| **בקאנד** | NestJS 11 — אימות Firebase Admin (Bearer), CORS, Firebase Auth Guard על routes מוגנים |
| **API מרכזי** | `GET /circles/nearby` (lat, lng, radiusKm), `POST /players/recalculate-rating` |
| **ריאל-טיים** | Firestore `onSnapshot` לרשימת מעגלים, מעגל בודד וצ'אט מעגל |
| **התראות** | expo-notifications, FCM — טוקן נשמר בפרופיל; מסך "התראות למנהל" |
| **מיקום** | expo-location — "השתמש במיקום הנוכחי" ביצירת מעגל + הרשאות ב־app.json |
| **פריסה** | בקאנד: Docker + Railway; קליינט: Expo (Build / EAS) |

## 5. מודל נתונים (ליבה)

- **UserProfile:** uid, email, displayName, photoURL, videoURL, level, gender, playStyle, role (player | manager), city, location (city, area, lat, lng), notificationSettings, stats (circlesJoined, circlesCreated, noShows, rating), profileCompleted.
- **Circle:** creatorId, title, location (name, lat, lng), city, area, courtType, dateTime, maxPlayers, players[], requiredLevel, status (open | full | closed | completed), genderRestriction, isUrgent, chatEnabled, createdAt.
- **Chat:** תת-אוסף `chats/{circleId}/messages` — senderId, senderName, text, createdAt.
- **Ratings:** אוסף `ratings` — circleId, raterId, ratedId, score; משמש לחישוב ושמירת `stats.rating`.

## 6. אבטחה ותנאי שימוש

- **Firestore:** רק משתמשים מחוברים; `users/{uid}` קריאה/כתיבה לבעלים; `circles` — קריאה לכולם, יצירה רק כיוצר; צ'אט — רק למשתתפי המעגל; דירוגים — יצירה וקריאה בהתאם.
- **בקאנד:** כל ה-routes המוגנים עוברים Firebase Auth Guard (אימות טוקן דרך Firebase Admin).

## 7. אופציות להתרחבות (Expansion)

### 7.1 הרחבה גאוגרפית
- **אירופה / דרום אמריקה:** פוטוולי פופולרי מאוד (ברזיל, ספרד, איטליה וכו'). התאמת שפה (אנגלית, פורטוגזית, ספרדית), אזורים וערים, ומטבע מקומי אם ייכנס מודל תשלום.
- **שוקי חוף נוספים:** איחוד האמירויות, יוון, קפריסין — כולם שווקים רלוונטיים לאותו use-case.

### 7.2 הרחבה ורטיקלית (ענפי ספורט / פעילויות)
- **ספורט חופים נוספים:** ביץ־וואלי, פוטבול חופים, אימונים קבוצתיים על החוף — אותו מנוע (מעגלים, דירוג, צ'אט) עם התאמת סוג פעילות.
- **טורנירים ואירועים:** יצירת אירועים רב-מעגלים, ליגות עונתיות, שיתוף עם מועדונים/מגרשים.

### 7.3 B2B ושותפויות
- **מגרשים ורשויות:** דף "מגרשים מומלצים", שיתוף פעולה עם רשויות לחופים (תל אביב, חיפה וכו') — פרסום מעגלים רשמיים או סבסוד.
- **מועדונים ומאמנים:** חשבון "ארגון/מועדון" — ניהול כמה מעגלים, רשימת מתאמנים, טורנירים פנימיים.
- **ספונסרים:** לוגו ותוכן ממותג במעגלים/אירועים (ציוד ספורט, משקאות, ביטוח).

### 7.4 מודל הכנסות (פוטנציאל)
- **Freemium:** שימוש בסיסי חינם; מנוי פרימיום — מעגלים ללא הגבלה, סטטיסטיקות מתקדמות, באדג'ים, הסרת פרסומות.
- **עמלות:** עמלה סמלית על מכירת כרטיסים למעגלים/טורנירים (אם יהפוך למודל תשלום).
- **פרסום:** באנרים/מודעות ממוקדות לפי אזור ורמה (ציוד, אירועים).
- **B2B:** דמי רישיון למגרשים/מועדונים או עמלה על הזרמת שחקנים.

## 8. שיפורים עתידיים (Roadmap)

### 8.1 חוויית משתמש ומוצר
- **חיפוש מתקדם:** סינון לפי סוג מגרש, מגדר, טווח שעות, "רק מעגלים עם מקום פנוי".
- **מפת מעגלים:** הצגת מעגלים על מפה (עם שמירה על פרטיות).
- **לוח שנה אישי:** אינטגרציה עם לוח השנה — מעגלים שהמשתמש הצטרף/יצר.
- **המלצות:** "מעגלים שמתאימים לך" לפי רמה, אזור והיסטוריה.
- **עיצוב ונגישות:** שיפור RTL, ניגודיות, תמיכה בגדלי טקסט, אנימציות עדינות.

### 8.2 חברה ו-reputation
- **תגובות מפורטות:** בנוסף לציון — תגובה טקסטואלית (אופציונלי) ו"תגים" (מגיע בזמן, כיף לשחק וכו').
- **היסטוריית משחקים:** דף "משחקים שעברו" עם רשימת משתתפים ודירוגים.
- **מערכת אמון:** סימון "שחקן אמין" / "מנהל פעיל" לפי קריטריונים אוטומטיים.

### 8.3 טכני ואמינות
- **Analytics ו־Monitoring:** אירועים (הצטרפות, יצירת מעגל, דירוג), מעקב אחר crashes ו־performance (למשל Expo / Sentry).
- **גיבוי ו־GDPR:** ייצוא נתונים למשתמש, מחיקת חשבון מסודרת, תיעוד מדיניות פרטיות.
- **שיפור API:** גרסה מסודרת ל־API, rate limiting, תיעוד (למשל OpenAPI) לשותפים עתידיים.
- **Offline:** תמיכה בסיסית ב־offline — צפייה במעגלים שנטענו, סנכרון כשחוזרים לרשת.

### 8.4 שיווק וצמיחה
- **הפניה (Referral):** "הזמן חבר — תקבל מעגל חינם" או יתרון פרימיום.
- **תוכן ושיווק:** בלוג/מדיה על פוטוולי, טיפים, סיפורי שחקנים — לשיפור SEO ורשתות חברתיות.
- **אינטגרציות:** שיתוף מעגל ל־WhatsApp/Instagram, כפתור "הוסף ללוח שנה".

### 8.5 מוניטיזציה (כשהמוצר בשל)
- **מנוי פרימיום:** כמתואר בסעיף ההתרחבות.
- **תשלום במעגלים:** הפקדה/תשלום למנהל מגרש או לארגון אירוע — עם עמלה לפלטפורמה.
- **טורנירים בתשלום:** רישום וקבלת פרסים/מדליות.

## 9. סיכום והמלצות להצגה למשקיעים

1. **להדגיש:** פלטפורמה ראשונה בישראל לפוטוולי חופים, עם מוצר עובד (אפליקציה + בקאנד + Firebase), דירוג שחקנים וצ'אט — יתרון ברור על קבוצות וואטסאפ/פייסבוק.
2. **לכלול במצגת:** סלידים על הבעיה, הפתרון, מסך הבית והמעגל (עם RTL ועברית), נתוני סטק, ושביל צמיחה — ישראל → הרחבה גאוגרפית ו־B2B.
3. **להזכיר אופציות התרחבות:** שווקים נוספים, ענפי ספורט חופים, טורנירים, שותפויות מגרשים/רשויות, ומודל פרימיום/עמלות/פרסום.
4. **לצרף Roadmap:** שיפורים עתידיים (חיפוש, מפה, המלצות, analytics, מוניטיזציה) כ־"Phase 2 / 3" לפי עדיפות.
